# 任务包 09：Orchestrator、RiskAgent 与合规强化

## 目标

强化最终研究结果的仲裁逻辑、风险降级、信息不足规则和违规表达拦截，保证系统始终是研究辅助工具，而不是交易建议工具。

## 范围

需要做：

- Orchestrator 三层流程：
  - 数据有效性检查。
  - 多 Agent 冲突处理。
  - 合规输出控制。
- 实现动态权重和时效性衰减基础逻辑。
- 实现分歧度输出：
  - 财务与技术冲突。
  - 行业与公司财务冲突。
  - 舆情与风险冲突。
  - 资金与基本面冲突。
- 实现 RiskAgent 一票降级：
  - RiskAgent 偏空或谨慎。
  - 置信度达到阈值。
  - 风险类型属于重大财务、监管、退市、诉讼、立案调查、重大违约、业绩暴雷、大股东爆仓等。
- 实现信息不足规则：
  - 关键数据缺失。
  - 只有 C/D 级证据。
  - 7 个 Agent 中超过 3 个失败。
  - RiskAgent 未运行成功。
  - 数据明显过期。
  - 多空分歧极高且无高可信证据支撑。
- 强化 ComplianceGuard：
  - 拦截买入、卖出、强烈买入、满仓、梭哈、必须持有、仓位、目标价、保证收益、一定上涨、无风险、稳赚等表达。
  - 替换为研究辅助表达。

不需要做：

- 不输出交易执行建议。
- 不做券商接入。
- 不做自动调仓。

## 验收标准

- RiskAgent 高置信风险时，最终研究分级自动降级。
- 违规表达在最终输出前被拦截或替换。
- 证据不足时返回 `信息不足，无法形成建议。`
- 高分歧时不强行输出单一强方向。
- 合规测试覆盖 forbidden phrases。
- 所有最终输出包含免责声明。
