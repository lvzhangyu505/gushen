# 任务包 10：日报、提醒与历史复盘闭环

## 目标

从“单次分析工具”升级为“长期投研工作台”：围绕自选股生成日报、触发提醒、保存历史分析并进行复盘。

## 范围

需要做：

- 实现 watchlist CRUD。
- 实现每日自选股扫描任务，MVP 可用 APScheduler。
- 生成每日投研日报：
  - 新增公告。
  - 新闻变化。
  - 行情/成交变化。
  - 风险变化。
  - 需要重新分析的股票。
- 实现触发式提醒：
  - 重大公告。
  - 财报或业绩预告。
  - 股价异常波动。
  - 成交量异常放大。
  - 资金连续流入或流出。
  - 行业政策变化。
  - 舆情突然升温或转负。
  - 风险新闻出现。
  - 原研究结论失效条件触发。
- 保存历史分析记录。
- 实现复盘记录：
  - 后续价格变化。
  - 是否触发失效条件。
  - 原判断是否有效。
  - 误判原因。
  - 哪个 Agent 贡献最大。
  - 哪个 Agent 噪音最大。
  - 是否需要调整权重。
- 前端增加或完善日报、风险中心、历史复盘视图。

不需要做：

- 不做全市场自动扫股。
- 不做组合管理。
- 不做自动调仓。
- 不做用户登录体系。

## 验收标准

- 自选股可以添加、编辑、删除。
- 可以生成某日期的日报。
- 风险提醒可以创建、查询、标记已读或处理。
- 可以查看某只股票历史分析记录。
- 可以创建复盘记录。
- 周复盘报告能列出本周判断有效和失效情况。
- 所有日报和复盘输出包含免责声明。
