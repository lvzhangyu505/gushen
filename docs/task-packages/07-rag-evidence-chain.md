# 任务包 07：RAG 与证据链

## 目标

建立文档入库、切分、向量化、检索、证据等级和前端证据链展示，让研究结论有可追溯依据。

## 范围

需要做：

- 建立 `documents` 和 `document_chunks` 数据模型。
- 支持文档类型：
  - 上市公司公告。
  - 定期报告摘要。
  - 监管文件。
  - 官方政策。
  - 授权财经新闻。
  - 行业资料或研报摘要。
  - 用户手动上传或录入材料。
- 建立证据等级：
  - S：交易所公告、上市公司公告、定期报告、监管文件。
  - A：官方政策、权威财经媒体、行业协会数据。
  - B：券商研报摘要、机构观点、行业数据库。
  - C：普通财经新闻、互动平台、社交舆情。
  - D：论坛传闻、短视频观点、未经验证消息。
- D 级证据不得作为最终研究结论依据。
- 只有 C/D 级证据时必须输出信息不足。
- 增加 embedding provider 抽象，不写死具体模型。
- 实现 Agent 级检索，每个 Agent 最多返回 3 条核心证据和 3 条反方证据。
- Orchestrator 只读取 Agent 精选证据和结构化 signal。
- 前端支持证据展开和来源展示。

不需要做：

- 不接付费受限数据。
- 不生成确定性投资建议。
- 不做复杂知识图谱。

## 验收标准

- 文档可以入库、去重、切分。
- chunk 可以写入 pgvector。
- Agent 可以检索到相关证据。
- 每个 Agent 返回证据数量受限。
- 证据展示包含来源、类型、等级、时间、摘要、相关性。
- 只有低等级证据时，系统不形成稳定研究判断。
- RAG 测试验证 unsupported suggestion 被拒绝。
