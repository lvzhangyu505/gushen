# PRD: Gushen AI 个人投研辅助工作台

更新时间：2026-07-03

## 1. 产品定位

Gushen 是一个面向个人投资者的 AI 股票研究工作台。系统围绕用户自选股和单只股票查询，收集合法市场信息、上市公司公告、财经新闻、政策变化、行业信息、财务数据、研报摘要和用户手动录入信息，通过多 Agent 分析、证据链追踪、风险校验、分歧解释与历史复盘，辅助用户形成更清晰的研究判断。

Gushen 不是券商交易系统，不是自动交易系统，也不替用户做最终投资决策。

统一免责声明：

> 本系统仅用于个人研究辅助，不构成确定性投资承诺。

## 2. 目标用户

- 个人投资者：希望把分散信息整理为可复盘的投研工作台。
- 深度自选股研究用户：每天跟踪少量重点股票的基本面、技术面、资金面、情绪面和风险面。
- 复盘型用户：记录自己的研究判断、实际操作笔记和后续结果，用于提升研究纪律。

## 3. 核心目标

- 聚合自选股相关事实信息，并保留原始来源元数据。
- 将事实、分析、建议、风险、证据和免责声明分字段输出。
- 通过 7 个专家 Agent 生成标准 `ExpertSignal`。
- 通过 Orchestrator 合成信号、解释分歧、校验证据和风险。
- 通过 RiskAgent 对重大风险执行一票降级。
- 通过 ComplianceGuard 拦截违规表达和无证据建议。
- 生成每日投研日报、触发式风险提醒和历史复盘记录。
- 支持基于 RAG 的搜索问答，证据不足时明确返回不足。

## 4. 默认研究范围

MVP 阶段支持：

- A 股。
- 单只股票查询。
- 用户手动输入股票代码或股票名称。
- 用户自选股研究。
- 事件驱动型研究。
- 日报、提醒和复盘围绕用户自选股展开。

MVP 阶段暂不支持：

- 港股、美股、基金、期货、期权、外汇、加密资产。
- 全市场自动扫股。
- 高频交易、日内盘口交易、自动组合调仓或自动跟单。

## 5. 分析周期

每次分析必须声明分析周期，不同周期不能混用。

| 周期 | 时间范围 | 适用问题 |
|---|---:|---|
| 短线 | 3-10 个交易日 | 量价、资金、舆情、短期事件 |
| 中期 | 1-3 个月 | 行业景气、财报预期、政策影响 |
| 长期 | 6 个月以上 | 公司基本面、产业趋势、长期风险 |

所有 Agent Signal 必须包含 `time_horizon` 字段。Orchestrator 最终输出必须说明当前判断主要适用于哪个周期。

## 6. 核心流程

### 6.1 单只股票分析

```text
用户输入股票名称或代码
  -> 创建 analysis_task
  -> 读取股票基础信息、行情快照、公告、新闻、财务、行业、舆情、风险数据
  -> 7 个专家 Agent 并行或准并行输出 ExpertSignal
  -> Orchestrator 聚合信号、处理冲突、计算分歧度、触发风险校验
  -> ComplianceGuard 校验输出
  -> 前端展示研究卡、专家信号表、证据链、风险提醒、缺失信息和免责声明
  -> 保存历史分析记录
  -> 后续进入复盘系统
```

### 6.2 自选股日报

系统定时扫描自选股，抓取公告、新闻、行情、资金、行业、政策、舆情变化，识别重大变化，生成每日投研日报。日报必须包含事实、分析、建议、风险、证据和免责声明。

### 6.3 触发式提醒

触发条件包括重大公告、财报或业绩预告、股价异常波动、成交量异常放大、资金连续流入或流出、行业政策变化、舆情突然升温或转负、风险新闻出现、原研究结论失效条件被触发。

## 7. 功能范围

### 7.1 自选股池

- 用户手动添加、移除、分组和备注关注股票。
- 支持股票代码、股票名称、市场、行业、关注理由、标签。
- MVP 默认覆盖 A 股，预留扩展字段但不承诺其他市场支持。

### 7.2 数据采集与证据沉淀

- 只使用合法公开 API、授权数据源、用户提供文件或手动录入信息。
- 不绕过付费墙，不抓取受限系统，不伪造来源。
- 每条来源数据必须记录来源、发布时间、采集时间、原文 URL、授权状态、可信度和原始元数据。
- 原始来源元数据必须先于 AI 分析持久化。
- 检索证据必须与模型生成分析分开存储。

### 7.3 个股研究卡

- 展示基本面、技术面、资金面、情绪面、风险面分析。
- 展示 Orchestrator 研究分级、分歧解释、缺失信息、失效条件和下一步观察点。
- 每个关键结论必须附带证据来源。
- 无证据时不得生成确定性判断。
- 页面必须展示统一免责声明。

### 7.4 研究辅助建议

允许输出的建议标签仅限：

- `值得关注`
- `继续观察`
- `风险偏高`
- `暂不建议介入`
- `等待确认信号`

每条建议必须附带证据、分析依据和风险说明。证据不足时返回：`信息不足，无法形成建议。`

### 7.5 风险提醒

系统识别并提醒：业绩下滑、监管处罚、重大诉讼、股东减持、股权质押、舆情异常、股价异动、财务异常、行业政策不利变化、退市风险、立案调查、重大违约和业绩暴雷。

### 7.6 投资笔记与复盘

- 用户可记录自己的判断、关注逻辑、实际操作和复盘结果。
- 实际操作仅作为个人笔记，不连接券商账户，不触发交易。
- 每次分析需要保存当时价格、分析周期、7 个 Agent Signal、Orchestrator 分级、核心证据、反方证据、主要风险、缺失信息、失效条件和后续观察点。

### 7.7 搜索问答

- 基于 RAG 检索回答自选股和市场问题。
- 回答必须分为 `facts`、`analysis`、`suggestions`、`risks`、`evidence`、`disclaimer`。
- 回答必须展示证据来源。
- 无足够证据时返回：`信息不足，无法形成建议。`

## 8. 证据等级

| 等级 | 来源类型 | 使用规则 |
|---|---|---|
| S | 交易所公告、上市公司公告、定期报告、监管文件 | 可作为主要结论依据 |
| A | 官方政策、权威财经媒体、行业协会数据 | 可作为主要结论依据 |
| B | 券商研报摘要、机构观点、行业数据库 | 可作为辅助结论依据 |
| C | 普通财经新闻、互动平台、社交舆情 | 只能作为情绪或事件参考 |
| D | 论坛传闻、短视频观点、未经验证消息 | 不得作为研究结论依据 |

主要结论只能基于 S/A/B 级证据。只有 C/D 级证据时，必须输出信息不足。

## 9. 非功能需求

- 可追溯：所有 AI 分析都能回溯到来源文档和证据项。
- 可审计：保存采集任务、分析任务、模型输出、合规审查和复盘记录。
- 可替换：数据源、LLM、embedding 和 reranker provider 必须可替换。
- 可扩展：MVP 使用 APScheduler，后续可迁移 Celery。
- 安全：不存储券商账户、交易密码、资金账户或下单凭证。
- 可靠：Agent 失败时返回部分结果并标记缺失，不能让页面崩溃。

## 10. 合规边界

禁止实现或暗示以下能力：

- 券商账户接入。
- 交易 API。
- 下单、撤单、自动交易。
- 账户同步、资金托管、资金转账。
- 自动组合调仓或自动跟单。
- 仓位建议，例如“建议仓位 30%”。
- 明确买入点位、卖出点位或目标价承诺。
- “必须买入”“必须卖出”“保证收益”“无风险”“稳赚”等确定性表述。

所有建议、报告、个股研究卡和问答结果页面必须展示：

> 本系统仅用于个人研究辅助，不构成确定性投资承诺。

## 11. MVP 验收标准

- 用户可以维护一个 A 股自选股池。
- 系统可以录入或采集至少三类来源文档：公告、新闻、政策或财报摘要。
- 系统可以创建 `analysis_task`，并返回 7 个专家 Agent 的标准 `ExpertSignal`。
- Orchestrator 输出研究分级、周期、综合方向、分歧度、关键证据、关键风险、缺失信息和下一步观察点。
- RiskAgent 高置信重大风险触发时，最终结果自动降级。
- ComplianceGuard 能拦截违规交易表达和无证据建议。
- 个股研究卡生成结果包含事实、分析、建议、风险、证据和免责声明。
- 无证据时返回：`信息不足，无法形成建议。`
- 建议标签仅使用允许枚举。
- 日报可以按日期和类型查询。
- 风险提醒可以创建、查询和标记状态。
- 搜索问答可以引用证据回答。
- UI 中建议、日报、个股研究卡和问答结果展示免责声明。
- 代码和接口中不存在交易、账户同步、下单、撤单或资金托管能力。
