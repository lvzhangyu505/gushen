# PRD: AI 投研信息与投资建议平台

## 1. 产品定位

本产品是个人自用的股票研究助手，定位为"信息收集 + 投资研究 + 决策辅助"平台。系统帮助用户每天自动收集 A 股市场信息、上市公司公告、财经新闻、政策变化、行业动态、财报数据、研报摘要和自选股相关信息，并通过 AI 进行结构化分析，生成投资研究建议、风险提醒和每日投研日报。

本产品不是券商交易系统，不提供自动买卖功能，不接入任何证券账户，不下单，不撤单，不托管任何资金账户。

## 2. 目标用户

- 个人投资者：希望把分散信息整理为可复盘的投研工作台。
- 深度自选股研究用户：每天跟踪少量重点股票的基本面、技术面、资金面、情绪面和风险面。
- 复盘型用户：记录自己的判断、实际操作和后续结果，用于提升研究纪律。

## 3. 核心目标

- 自动聚合自选股相关事实信息。
- 将原始信息结构化为事实、模型分析、投资建议、风险提示和证据链。
- 生成每日早报、收盘复盘、夜间深度总结。
- 为每只自选股生成可追溯的研究卡。
- 支持自然语言问答，例如"某只股票最近发生了什么"、"今天哪些政策影响新能源"、"我的自选股里谁风险最高"。

## 4. 功能范围

### 4.1 自选股池

- 用户手动添加、移除、分组和备注关注股票。
- 支持股票代码、股票名称、市场、行业、关注理由、标签。
- MVP 默认覆盖 A 股，预留港股和全球市场扩展字段。

### 4.2 市场信息采集

- 自动采集指数、板块、新闻、公告、政策、财报、研报摘要。
- 数据源优先使用合法公开 API、授权数据源或用户手动导入。
- 所有采集内容必须记录来源、发布时间、采集时间、原文 URL、授权状态和可信度。

### 4.3 个股研究卡

- 对每只自选股生成基本面、技术面、资金面、情绪面、风险面分析。
- 每个分析结论必须附带证据来源。
- 无证据时不得生成确定性判断。

### 4.4 投资研究建议

- 允许输出的建议标签仅限：
  - 值得关注
  - 继续观察
  - 风险偏高
  - 暂不建议介入
  - 等待确认信号
- 不输出强制买入、卖出、加仓、清仓、下单、撤单等交易指令。
- 每条建议必须附带信息来源、分析依据、风险说明。

### 4.5 风险提醒

系统识别并提醒：

- 业绩下滑
- 监管处罚
- 重大诉讼
- 股东减持
- 股权质押
- 舆情异常
- 股价异动
- 财务异常
- 行业政策不利变化

### 4.6 投资笔记

- 用户记录自己的判断、关注逻辑、实际操作和复盘结果。
- 笔记可绑定股票、日期、建议、日报或风险提醒。
- 用户的实际操作仅作为个人笔记，不触发交易。

### 4.7 搜索问答

- 基于 RAG 检索回答自选股和市场问题。
- 回答必须展示证据来源。
- 无足够证据时返回"信息不足，无法形成建议。"

## 5. 非功能需求

- 可追溯：所有 AI 分析都能回溯到来源文档。
- 可审计：保存采集任务、生成任务和模型输出记录。
- 可替换：数据源、LLM 和 embedding provider 必须可替换。
- 可扩展：MVP 使用 APScheduler，后续可迁移 Celery。
- 安全：不存储券商账户、交易密码、资金账户或下单凭证。

## 6. 合规边界

- 禁止接入券商接口。
- 禁止自动交易。
- 禁止代客理财表述。
- 禁止确定性收益承诺。
- 禁止在 UI 中隐藏免责声明。
- 禁止使用未授权数据源或绕过付费墙。

所有建议和报告页面必须展示：

> 本系统仅用于个人研究辅助，不构成确定性投资承诺。

## 7. MVP 验收标准

- 用户可以维护一个 A 股自选股池。
- 系统可以录入或采集至少三类来源文档：公告、新闻、政策或财报摘要。
- 系统可以为自选股生成研究卡，且每个结论都有证据。
- 系统可以生成每日早报、收盘复盘、夜间深度总结模板内容。
- 系统可以创建风险提醒。
- 搜索问答可以基于已入库证据回答问题。
- 所有建议页面显示免责声明。
- 代码和接口中不存在交易、账户同步、下单、撤单或资金托管能力。
