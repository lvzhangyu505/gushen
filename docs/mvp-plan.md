# MVP 开发计划

## 1. MVP 范围

首期覆盖 A 股自选股投研助手：

- 自选股池
- 来源文档入库
- RAG 检索
- 个股研究卡
- 风险提醒
- 每日投研日报
- 投资笔记
- 搜索问答
- 数据源状态

不包含：

- 券商账户接入
- 自动交易
- 下单、撤单
- 资金账户托管
- 多用户商业化权限体系

## 2. 里程碑

### M1: 文档和工程骨架

- 完成 `docs/` 文档和 `AGENTS.md`。
- 初始化 `frontend/`、`backend/`、`infra/`、`scripts/`。
- 配置 PostgreSQL + pgvector Docker Compose。

### M2: 后端基础能力

- FastAPI 项目结构。
- 数据库迁移。
- 自选股 CRUD。
- 来源文档入库。
- 采集任务状态。

### M3: RAG 和 Agent MVP

- 文档分块和 embedding。
- 证据检索。
- 个股研究卡生成。
- 合规审查。
- 建议证据链保存。

### M4: 日报和风险中心

- 早报、收盘复盘、夜间深度总结生成。
- 风险扫描。
- 风险提醒状态流转。

### M5: 前端 MVP

- 仪表盘。
- 自选股。
- 个股研究。
- 日报。
- 风险中心。
- 投资笔记。
- 搜索问答。
- 数据源状态。

### M6: 验收与加固

- 文档验收。
- 后端单元测试。
- RAG 证据测试。
- 前端免责声明测试。
- 合规扫描。

## 3. 验收标准

- 自选股可以添加、编辑、删除。
- 来源文档可以入库、去重、分块、向量化。
- 个股研究卡生成结果包含事实、分析、建议、风险、证据和免责声明。
- 无证据时返回"信息不足，无法形成建议。"
- 建议标签仅使用允许枚举。
- 日报可以按日期和类型查询。
- 风险提醒可以创建、查询和标记状态。
- 搜索问答可以引用证据回答。
- UI 中建议和报告页面展示免责声明。
- 仓库中不存在交易或券商账户能力。

## 4. 默认技术选择

- APScheduler 作为 MVP 调度器。
- pgvector 作为向量数据库。
- 数据源通过 adapter 抽象。
- LLM 和 embedding 通过 provider-neutral service 抽象。
